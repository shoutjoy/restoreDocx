## 1. DOCX란 무엇인가

`.docx` 파일은 **ZIP 압축 아카이브**입니다. 확장자만 Word 문서처럼 보일 뿐, 내부는 XML·이미지·관계(rels) 파일들의 묶음입니다.

| 구분 | 설명 |
|------|------|
| 확장자 | `.docx` |
| 실제 형식 | ZIP (PK 헤더 `50 4B 03 04`) |
| 본문 위치 | `word/document.xml` |
| 표준 | Office Open XML (OOXML) |

Word에서 문서를 열면 ZIP을 풀고 `word/document.xml`을 렌더링합니다. 이 XML이 손상되거나 ZIP 구조가 깨지면 Word는 “파일을 열 수 없음” 오류를 냅니다. **본문 데이터가 XML 조각으로 남아 있으면** 이 도구로 텍스트만이라도 뽑아낼 수 있는 경우가 많습니다.

## 2. 내부 폴더·파일 구조

일반적인 최소 DOCX 패키지 구조는 다음과 같습니다.

```
example.docx  (ZIP)
├── [Content_Types].xml      ← 각 파트의 MIME 타입 정의
├── _rels/
│   └── .rels                ← 패키지 루트 관계 (document.xml 연결)
└── word/
    └── document.xml         ← ★ 본문(단락·글자·표 등)이 들어 있는 핵심 파일
```

### 주요 파일 설명

| 경로 | 역할 |
|------|------|
| `[Content_Types].xml` | `word/document.xml` 등 각 파트의 Content-Type 선언 |
| `_rels/.rels` | 루트에서 `word/document.xml`로 가는 Relationship |
| `word/document.xml` | WordprocessingML 본문. 단락·런·텍스트가 XML로 기록됨 |

실제 Word 문서에는 아래 파일들도 자주 포함됩니다.

```
├── word/
│   ├── styles.xml           ← 스타일 정의
│   ├── settings.xml         ← 문서 설정
│   ├── fontTable.xml        ← 글꼴
│   ├── numbering.xml        ← 번호 매기기
│   └── media/               ← 삽입 이미지
├── docProps/
│   ├── core.xml             ← 제목, 작성자 등 메타데이터
│   └── app.xml
└── word/_rels/document.xml.rels  ← 본문과 이미지·헤더 등의 관계
```

이 복구 도구는 **본문 복원**에 집중하므로, 추출·재생성 시에는 주로 `document.xml`과 패키지를 열기 위한 최소 파일(`[Content_Types].xml`, `_rels/.rels`)만 사용합니다.

### document.xml 안의 텍스트 표현

본문은 WordprocessingML 네임스페이스 `http://schemas.openxmlformats.org/wordprocessingml/2006/main` 아래에 정의됩니다.

| XML 요소 | 의미 |
|----------|------|
| `<w:document>` | 문서 루트 |
| `<w:body>` | 본문 영역 |
| `<w:p>` | 단락(Paragraph) |
| `<w:r>` | 런(Run, 동일 서식의 텍스트 묶음) |
| `<w:t>` | 실제 글자(Text) |

예시:

```xml
<w:document xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main">
  <w:body>
    <w:p>
      <w:r>
        <w:t>안녕하세요.</w:t>
      </w:r>
    </w:p>
  </w:body>
</w:document>
```

도구는 `<w:p>` 단위로 순회하며 각 단락 안의 `<w:t>` 내용을 이어 붙이고, 단락마다 줄바꿈을 넣어 읽기 쉬운 평문으로 만듭니다.

## 3. 문서가 깨지는 대표적인 경우

| 상황 | 증상 | 이 도구에서의 처리 |
|------|------|-------------------|
| ZIP 헤더 손상 | Word·압축 프로그램 모두 열기 실패 | **강제 추출 모드**: 파일을 텍스트로 읽고 `<w:t>...</w:t>` 정규식 매칭 |
| `.docx`를 메모장 등으로 저장 | XML·바이너리가 한 줄 텍스트로 보임 | 동일하게 강제 추출 모드 |
| `word/document.xml`만 손상 | ZIP은 열리나 Word 오류 | ZIP 모드에서 XML 파싱 실패 시 부분 추출 가능성 있음 |
| 정상 DOCX | Word에서 정상 열림 | ZIP 모드로 정상 추출 (참고·백업용으로도 사용 가능) |

`.doc`(구형 바이너리) 형식은 **지원하지 않습니다**. `.docx`, `.zip`, 구조가 노출된 `.txt`만 대상입니다.

## 4. 이 도구의 동작 원리

```
[파일 업로드]
      │
      ▼
 ArrayBuffer로 읽기
      │
      ▼
 JSZip으로 ZIP 파싱 시도 ──성공──► processValidZip()
      │                              │
      │                              ├─ ZIP 내부 파일 목록 표시
      │                              ├─ word/document.xml 로드
      │                              └─ DOMParser로 <w:p>/<w:t> 추출
      │
     실패
      │
      ▼
 processCorruptedText()  (텍스트 강제 추출)
      │
      ├─ 정규식: /<w:t[^>]*>([\s\S]*?)<\/w:t>/g
      ├─ XML 엔티티 디코딩 (&amp; → & 등)
      └─ 태그 없으면 원시 텍스트 전체 출력
```

### 복원(다운로드) 시 생성하는 최소 DOCX

「DOCX 다운로드」를 누르면 JSZip으로 아래 3개 파일만 넣은 새 `.docx`를 만듭니다.

1. `[Content_Types].xml`
2. `_rels/.rels`
3. `word/document.xml` — 추출한 평문을 줄 단위 `<w:p>`로 변환

**서식·이미지·표·머리글/바닥글은 복원되지 않습니다.** 텍스트만 담긴 새 Word 파일입니다.

## 5. 복구 절차 (단계별)

### 5.1 사전 준비

1. **원본 파일 복사본**을 만듭니다. (`문서_손상.docx` → `문서_손상_백업.docx`)
2. Word에서 「다른 이름으로 저장」이 되는지 한 번 시도합니다. 되면 그 복사본을 우선 사용합니다.
3. 이 도구는 **브라우저에서만** 동작하며, 파일을 서버로 업로드하지 않습니다 (로컬 처리).

### 5.2 도구로 복구하기

| 단계 | 작업 | 기대 결과 |
|------|------|-----------|
| 1 | 앱 실행 후 손상 파일을 드래그하거나 선택 | 상태 메시지 표시 |
| 2 | 「정상적인 압축 구조」 메시지 확인 | ZIP 모드: 내부 구조 탭에 `word/document.xml` 등 목록 |
| 3 | 「압축 헤더가 손상」 경고 시 | 강제 추출 모드로 전환됨 — 계속 진행 |
| 4 | **복원된 텍스트** 탭에서 내용 확인 | 본문이 맞는지 눈으로 검증 |
| 5 | **document.xml 원본** 탭 | XML이 남아 있는지, 손상 지점 확인 |
| 6 | **파일 내부 구조** 탭 | `document.xml` 존재 여부 확인 |
| 7 | 「텍스트 복사」 또는 「DOCX 다운로드」 | Word·메모장 등에 붙여넣기 또는 새 docx 저장 |
| 8 | Word에서 다운로드한 파일 열기 | 추가 편집·서식 재적용 |

### 5.3 추출 결과가 비어 있거나 이상할 때

1. **document.xml 원본** 탭에 `<w:t>` 또는 한글/영문 본문이 보이는지 확인합니다.
2. 보이는데 추출만 안 되면 → 브라우저 콘솔(F12) 오류 확인 후, [수동 복구](#9-수동-복구-고급)를 시도합니다.
3. 원본 탭에도 의미 있는 XML이 없으면 → 디스크 복구·이전 버전·OneDrive/SharePoint 버전 기록 등 **다른 백업**을 찾아야 합니다.

### 5.4 복구 후 권장 작업

- 추출 텍스트를 **새 Word 문서**에 붙여넣고 제목·스타일을 다시 적용합니다.
- 중요 문서는 복구본과 원본 손상 파일을 **둘 다 보관**합니다.
- 「DOCX 다운로드」로 만든 파일은 최소 구조이므로, 필요 시 Word에서 「다른 이름으로 저장」으로 정식 docx로 다시 저장합니다.

## 6. 앱 실행 방법

별도 설치·빌드 과정이 없습니다. `index.html` 하나로 동작합니다.

### 방법 A: 파일 더블클릭

1. 프로젝트 폴더에서 `index.html`을 더블클릭합니다.
2. 기본 브라우저(Chrome, Edge 권장)에서 열립니다.

### 방법 B: 로컬 HTTP 서버 (선택)

일부 환경에서 `file://` 제약이 있을 때:

```powershell
cd c:\CusorApps\msword_restore
python -m http.server 8080
```

브라우저에서 `http://localhost:8080` 접속.

### 외부 의존성 (CDN)

| 라이브러리 | 용도 |
|------------|------|
| Tailwind CSS | UI 스타일 |
| JSZip 3.10.1 | ZIP/DOCX 읽기·쓰기 |

오프라인 사용 시에는 해당 CDN 스크립트를 로컬로 받아 `index.html` 경로를 수정해야 합니다.

## 8. 한계와 주의사항

- **텍스트만** 복구합니다. 표, 그림, 차트, 수식, 주석, 변경 내역 추적 등은 복원하지 않습니다.
- 강제 추출 모드에서는 단락 구조가 깨져 **줄바꿈이 부정확**할 수 있습니다.
- 개인정보·기밀 문서도 브라우저 메모리에서만 처리되지만, **공용 PC**에서는 사용 후 탭을 닫는 것이 좋습니다.
- 완전히 덮어쓰인 파일(0바이트, 다른 형식으로 전환)은 복구 불가능할 수 있습니다.

## 9. 수동 복구 (고급)

ZIP이 어느 정도 살아 있을 때:

1. 파일 확장자를 `.docx` → `.zip`으로 변경합니다.
2. 7-Zip 등으로 압축을 풉니다.
3. `word/document.xml`을 메모장·VS Code로 엽니다.
4. `<w:t>` 태그 사이의 문자열을 수동으로 복사하거나, 이 도구에 같은 파일을 넣어 자동 추출합니다.

ZIP이 완전히 깨졌을 때:

1. 메모장++ 등으로 파일을 **텍스트**로 엽니다.
2. `document.xml` 또는 `<w:t` 문자열이 보이면, 그 파일을 이 도구의 **강제 추출 모드**(.txt로 저장해도 됨)에 넣습니다.
